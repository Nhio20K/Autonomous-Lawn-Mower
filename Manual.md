# 🚜 ส่วนที่ 1: คู่มือผู้ควบคุมระบบหน้างาน (Operator Manual)

คู่มือสำหรับการใช้งานทั่วไปในแต่ละวัน การควบคุมรถตัดหญ้าผ่านโหมดต่างๆ และการดูแลรักษาความปลอดภัยขั้นต้น

## 1. การควบคุมผ่านรีโมทคอนโทรล
* **1.1** สวิตช์ที่ 1 ใช้สำหรับสลับโหมด
* **1.2** สวิตช์ที่ 2 ใช้สำหรับสสตาร์ทเครื่องยนต์
* **1.3** คันโยกที่ 1 ใช้สำหรับเร่งเครื่องยนต์
* **1.4** คันโยกที่ 2 ใช้สำหรับควบคุมทิศทางของรถ

## 2. การชาร์จแบตเตอรี่
* **2.1** เสียบสายชาร์จเข้ากับช่องรับสายชาร์จของตัวเครื่อง
* **2.2** กด ปุ่มหยุดฉุกเฉิน ค้างไว้ และเปิด สวิตช์ Power
* **2.3** สังเกตไฟสถานะที่เครื่องชาร์จจะแสดงเป็น สีแดง และตัวเลขบอกค่าแรงดันไฟฟ้าที่หน้าจอมิเตอร์จะเพิ่มสูงขึ้น
* **2.4** เมื่อชาร์จไฟจนเต็มระบบ ไฟสถานะที่เครื่องชาร์จจะเปลี่ยนเป็น สีเขียว

## 3. การทำงานแบบ Manual mode
* **3.1** หมุนเพื่อปลด ปุ่มหยุดฉุกเฉิน
* **3.2** เปิดสวิตช์การทำงานหลัก เพื่อจ่ายกระแสไฟให้ระบบควบคุมและชุดขับเคลื่อน
* **3.3** เปิดสวิตช์กล่องควบคุมและเปิดการทำงานของรีโมท
* **3.4** สังเกตุไฟสถานะที่กล่องควบคุมต้องไม่ติด (Manual mode) ถ้าไฟสถานะติดให้โยกสวิตช์เลือกโหมดไปอีกด้าน
* **3.5** ทดสอบและบังคับทิศทางของตัวเครื่อง (เดินหน้า ถอยหลัง เลี้ยวซ้าย ขวา)

## 4. การทำงานแบบ Auto mode
* **4.1** ตรวจสอบให้แน่ใจว่าแบตเตอรี่มีพลังงานเพียงพอ และน้ำมันเชื้อเพลิงอยู่ในระดับที่เหมาะสม
* **4.2** วาง RTK Base Station ในตำแหน่งที่มีสัญญาณดาวเทียมชัดเจน และเปิดการทำงาน
* **4.3** เปิด Laptop และเชื่อมต่อ Wi-Fi เข้ากับระบบของรถ
* **4.4** เปิดโปรแกรม ROS2 บน Laptop ด้วยคำสั่ง:
  ```bash
  cd ros2_ws
  source install/setup.bash
  ./start_robot.sh
  ```
* **4.5** กำหนดขอบเขตพื้นที่ตัดหญ้า (Boundary) และ Waypoint บนโปรแกรม RViz
* **4.6** โยกสวิตช์เลือกโหมดไปที่ Auto mode สังเกตไฟสถานะที่กล่องควบคุมต้องติด
* **4.7** พิมพ์ “ go ” บนโปรแกรมเพื่อเริ่มการทำงานอัตโนมัติ
* **4.8** สังเกตการทำงานของรถ ระบบจะวิ่งตามเส้นทางที่กำหนดโดยอัตโนมัติ และหยุดเองเมื่อครบพื้นที่

## 5. ระบบความปลอดภัย
* **5.1** ปุ่มหยุดฉุกเฉิน (Emergency Stop) — กดปุ่มแดงบนตัวเครื่องเพื่อหยุดระบบทันทีในกรณีฉุกเฉิน
* **5.2** ระบบตรวจจับสิ่งกีดขวาง — LiDAR จะตรวจจับและหยุดรถอัตโนมัติเมื่อมีสิ่งกีดขวางในระยะ 1 เมตร
* **5.3** ระบบหยุดอัตโนมัติเมื่อออกนอกพื้นที่ที่กำหนด
* **5.4** ตรวจสอบพื้นที่ให้ปราศจากสิ่งกีดขวางขนาดเล็กที่ LiDAR อาจตรวจไม่พบ เช่น สายไฟ ก้อนหิน ก่อนเริ่มใช้งาน

## 6. การดับเครื่องและเก็บรักษา
* **6.1** โยกสวิตช์เลือกโหมดกลับไปที่ Manual mode
* **6.2** ปิดสวิตช์กล่องควบคุมและปิด Laptop
* **6.3** กดปุ่มหยุดฉุกเฉินเพื่อตัดกระแสไฟฟ้าทั้งหมด
* **6.4** ปิดสวิตช์ Power หลัก
* **6.5** ถอดสายชาร์จ (ถ้ามี) และทำความสะอาดตัวเครื่องหลังใช้งาน
* **6.6** เก็บรถในที่ร่ม แห้ง และปลอดภัยจากฝนและความชื้น

# 💻 ส่วนที่ 2: คู่มือการติดตั้งและการตั้งค่าระบบ (Developer Installation & Surveying Guide)

คู่มือขั้นตอนทางเทคนิคสำหรับเริ่มติดตั้งระบบบนคอมพิวเตอร์ใหม่ การจูนโครงสร้างพื้นฐาน และการสำรวจแนวเขตสนาม

## 1. ขั้นตอนการติดตั้งโปรแกรมเริ่มจาก Ubuntu 22.04 (Fresh Installation)

### ขั้นตอนที่ 1.1: วิธีการนำไฟล์โปรเจคเข้ามาในเครื่อง

#### 📂 วิธีที่ A: การดาวน์โหลดไฟล์บีบอัด (.ZIP)
1. ทำการบีบอัดโฟลเดอร์โปรเจค `ros2_ws` จากเครื่องทดสอบเดิมเป็นไฟล์ `ros2_ws.zip` (โดยลบโฟลเดอร์ที่ไม่จำเป็นอย่าง `build`, `install`, `log` ออกก่อนเพื่อลดขนาดไฟล์)
2. นำไฟล์ ZIP ไปคลายตัว (Extract) ลงบนเครื่องใหม่ไว้ที่พาธโฟลเดอร์หน้าหลัก (Home Directory): `/home/<ชื่อผู้ใช้ใหม่>/`
3. เปลี่ยนชื่อโฟลเดอร์ เช่น: `ros2_ws`

#### 📂 วิธีที่ B: การดึงผ่านระบบ Git:
```bash
sudo apt update && sudo apt install git -y
git clone https://github.com/Nhio20K/Autonomous-Lawn-Mower.git ~/ros2_ws
```

---

### ขั้นตอนที่ 1.2: การติดตั้ง ROS2 Humble และ Dependencies
หลังจากได้โฟลเดอร์ `~/ros2_ws` มาอยู่ในเครื่องแล้ว ให้รันสคริปต์ติดตั้งตามลำดับต่อไปนี้:

1. **เข้าโฟลเดอร์โปรเจค:**
   ```bash
   cd ~/ros2_ws
   ```
2. **จัดการสิทธิ์ในการเข้าใช้พอร์ต USB:**
   ```bash
   sudo usermod -aG dialout $USER
   ```
   *(หมายเหตุ: ต้องทำการปิด Terminal แล้วเปิดใหม่ หรือ Reboot เครื่อง เพื่อให้สิทธิ์มีผล)*
3. **ติดตั้งระบบปฏิบัติการหุ่นยนต์ ROS2 Humble:** ให้เปิดใช้งานสิทธิ์การรันสคริปต์ และสั่งรันตัวติดตั้งอัตโนมัติ:
   ```bash
   chmod +x install_ros2_humble.sh
   ./install_ros2_humble.sh
   ```
4. **ติดตั้งแพ็คเกจเสริมที่หุ่นยนต์ต้องใช้:**
   ```bash
   chmod +x install_dependencies.sh
   ./install_dependencies.sh
   ```
   *(สคริปต์นี้จะลงทะเบียนไลบรารีนำทาง Nav2, ตัวคำนวณ EKF, และไลบรารีใน Python เช่น numpy=1.23.5 และ shapely ให้พร้อมใช้ทันที)*  
   รายการLibrary ที่จำเป็น อยู่ในไฟล์ `~/ros2_ws/requirements.txt`
5. **คอมไพล์โค้ดโปรเจค (Colcon Build):**
   ```bash
   source /opt/ros/humble/setup.bash
   colcon build --symlink-install
   ```

### A. การระบุค่าพิกัดอ้างอิงและ NTRIP สำหรับ GPS RTK

## 3. การสำรวจขอบเขตสนามตัดหญ้า (Field Surveying Instruction)
นี่คือคู่มือในการพาหุ่นยนต์เดินสแกนเพื่อสร้างขอบเขตรั้วความปลอดภัย เพื่อไม่ให้ล้อหลุดตกลงไปในแปลงดอกไม้หรือสระน้ำ

### ขั้นตอนปฏิบัติหน้างาน:
1. **เตรียมหุ่นยนต์ให้พร้อม:** นำหุ่นไปวางในจุดเริ่มต้นที่เรียบในสนาม
2. **รันตัวคุมรั้วในโหมด Teach-in:**
   ```bash
   ros2 run robot_bridge geofence_enforcer
   ```
   *หน้าจอจะแสดงทางเลือก ให้พิมพ์เลือกหมายเลข **`2` (สร้างรั้วใหม่ / Teach-in)** แล้วกด Enter*
3. **การบังคับบันทึกจุด:**
   * ใช้คีย์บอร์ดหรือคอนโทรลเลอร์ควบคุมขับหุ่นยนต์ช้าๆ ไปยังมุมแรกของสนามหยุดนิ่ง แล้วกด **[Enter]** ที่ PC เพื่อบันทึกจุดมุมที่ 1
   * ขับต่อไปยังมุมถัดไปและกด **[Enter]** เพื่อบันทึกมุมถัดไปเรื่อยๆ จนกระทั่งเดินครบรอบรูปทรงสนาม
4. **การเซฟบันทึกขอบเขต:**
   * เมื่อวนครบรอบและเก็บจุดมุมทั้งหมดเรียบร้อยแล้ว ให้พิมพ์ตัวอักษร **`s`** ใน Terminal แล้วกด **[Enter]**
   * โปรแกรมจะประมวลผลเซฟพิกัดลงไฟล์ `lawn_geofence.yaml`
   * **กลไก Datum อัตโนมัติ:** ระบบจะตั้งจุดเริ่มต้นจุดแรกให้เป็นค่า Datum 0,0 ของพิกัดแผนที่ (Local map) โดยอัตโนมัติ ทำให้ผู้ใช้ไม่ต้องไปตั้งค่าเองเมื่อสลับสนามใหม่

## 4. 🚜 การเปิดใช้งานเครื่องและสั่งตัดหญ้าจริง (Mower Operations)
ขั้นตอนการรันโปรแกรมระบบอัตโนมัติทั้งหมดเพื่อปฏิบัติงานและบังคับการตัดหญ้าจริง

### ขั้นตอนปฏิบัติการ:
1. **รันคำสั่งเริ่มระบบหุ่นยนต์หลัก:**
   ```bash
   cd ~/ros2_ws
   ./start_robot.sh
   ```
   *สคริปต์จะทำการเปิดโปรแกรมทั้งหมดขึ้นมาแยกออกเป็น 6 แท็บโดยอัตโนมัติ พร้อมทั้งรันระบบ EKF Sensor Fusion เพื่อล็อกศูนย์ตำแหน่งกับจุด Datum ที่บันทึกไว้ในขั้นตอนสำรวจสนาม*
2. **การทำงานของระบบ Safety:**
   * ระหว่างที่หุ่นตัดหญ้าเคลื่อนที่ โหนด `geofence_enforcer` (แท็บที่ 4) จะใช้ไลบรารี `shapely` เปรียบเทียบตำแหน่งเมตร X, Y ของหุ่นเทียบกับแนวรั้วในไฟล์สนามจริงตลอดเวลาที่ความถี่ 10Hz
   * **หากหุ่นล้ำออกนอกเขต:** จะส่งสัญญาณ `1` ไปที่ Topic `/emergency_stop` ซึ่งจะทำให้โหนด `teleop_stm.py` บล็อกสัญญาณควบคุมความเร็วล้อ ล้อจะล็อกหยุดทำงานทันทีเพื่อความปลอดภัยสูงสุด
3. **การสั่งตัดหญ้าอัตโนมัติ (Mowing Execute):**
   * ไปที่แท็บชื่อ **`5. MOWING`**
   * รอจนกระทั่งระบบเช็คความมั่นใจพิกัดนิ่งดีแล้ว
   * พิมพ์คำสั่ง **`go`** แล้วกด **[Enter]**
   * หุ่นยนต์จะเริ่มกระบวนการคำนวณเส้นทางการเดินตัดหญ้าแบบซิกแซก (Coverage Path Planning) และเริ่มเคลื่อนไหวปฏิบัติงานโดยสมบูรณ์

# 🛠️ ส่วนที่ 3: คู่มือการจูนระบบและการวิเคราะห์สถาปัตยกรรมเชิงลึก (System Tuning & Deep Diagnostic Guide)

คู่มือวิชาการเชิงลึกสำหรับนักพัฒนา เพื่ออธิบายกลไกการรับส่งข้อมูลใน ROS2 การจูนพารามิเตอร์เซ็นเซอร์ และคำสั่งสำหรับการตรวจจับวิเคราะห์ข้อผิดพลาด

## 1. โครงสร้างการทำงานของพารามิเตอร์ใน ROS 2 (Launch vs. ไฟล์รอง Python)
**ลำดับความสำคัญของพารามิเตอร์** ที่ซับซ้อนในหุ่นยนต์คันนี้มีดังนี้:
* **ไฟล์หลัก (`hardware_bringup.launch.py`):** เป็นหน้าด่านหลักในการรันระบบ หากมีตัวแปรใดตั้งค่าไว้ในไฟล์นี้ **ระบบจะยึดถือค่านี่เป็นหลัก (Override) และเพิกเฉยค่าดั้งเดิมในไฟล์รองไปเลยทันที**
* **ไฟล์รอง (ไฟล์โค้ด Python/C++ เช่น `vision_node.py` หรือ `teleop_stm.py`):** เป็นจุดประกาศตัวแปรเริ่มต้น มีไว้เพื่อความปลอดภัย **"กันหลุด" (Safety Fallback)** หากไฟล์หลักไม่มีการประกาศค่าจูน หรือระบบสัญญาณหลุด โค้ดจะดึงค่าเริ่มต้นในนี้มาใช้นำทางเพื่อป้องกันหุ่นยนต์ชนอุบัติเหตุ
* **ข้อกำหนดสำคัญ:**
  * ตัวแปรใดที่ระบุว่า **[จูนผ่านไฟล์ Launch ได้]** ➡️ ให้แก้ไขที่ไฟล์ Launch เพียง **จุดเดียว** เท่านั้น ห้ามไปแก้ในไฟล์รอง (เพราะจะไม่มีผลอะไรเมื่อรันจริง)
  * ตัวแปรใดที่ระบุว่า **[จูนในไฟล์รอง (Python) เท่านั้น]** ➡️ แปลว่าตัวแปรนั้นไม่ได้ทำระบบรับค่าผ่าน ROS 2 Parameter ไว้ ผู้พัฒนาต้องเปิดไฟล์ Python เข้าไปแก้บรรทัดโค้ดโดยตรงจึงจะมีผลหน้างาน

## 3. ขอบเขตการทำงานของเลเซอร์วัดระยะ (LiDAR Field of View Boundary)
ระบบความปลอดภัยขั้นต้นใช้สแกนรอบทิศทาง **RPLiDAR A1** แต่เนื่องจากตัวบอร์ดติดตั้งมีลักษณะเฉพาะตัว ทำให้กลไกสแกนของหุ่นมีสูตรคำนวณขอบเขตด้านหน้าหุ่นดังนี้:  
พารามิเตอร์ของ LiDAR ถูกประกาศไว้ในฟังก์ชัน `__init__` ของโหนด `teleop_stm` :

1. **`self.stop_dist_lidar = 1.2` (ระยะหยุดเบรกฉุกเฉิน) [จูนในไฟล์รอง `teleop_stm.py` เท่านั้น]:**  
   ระยะห่างต่ำสุดในแนวเมตรที่ยิงไปชนวัตถุหน้าตรงแล้วจะหยุดฉุกเฉิน
2. **`self.safety_angle_rad = 0.785` (มุมจำกัดขอบสแกนหน้าตรง) [จูนในไฟล์รอง `teleop_stm.py` เท่านั้น]:**  
   ค่ามุมเรเดียน (+/- 45 องศา) รอบจุดศูนย์กลางหน้าหุ่นยนต์ (เนื่องจากตัวสแกนติดตั้งสลับหัว 180 องศาในคันจริง มุมหน้าตรงจึงครอบคลุม 135° ถึง 225° หรือใกล้เคียง π และ -π)
3. **`len(danger_points) >= 10` (ตัวกรอง Noise สัญญาณหลอก) [จูนในไฟล์รอง `teleop_stm.py` เท่านั้น]:**  
   LiDAR ต้องตรวจสะท้อนเจอวัตถุหน้าตรงพร้อมกันมากกว่า `10` จุดขึ้นไปในหนึ่งรอบสแกน ถึงจะยืนยันว่าเจออุปสรรคจริง ป้องกันปัญหารถเบรกหลอกเนื่องจากฝุ่น ควัน หรือมีใบหญ้าชูกลางพัดปลิวเข้ามาชนมุมกล้องชั่วขณะ
   * ⚠️ **หมายเหตุผู้พัฒนา:** ในโค้ด `teleop_stm.py` บรรทัดที่ 769 มีคอมเมนต์ระบุว่าให้ปรับเป็น `25` เพื่อลดปัญหาเศษหญ้าปลิวผ่าน แต่ตัวเลขรันจริงยังเป็น `10` หากหน้างานเกิดรถเบรกหลอกบ่อยจากเศษฝุ่นหรือหญ้า ให้ผู้พัฒนาระบบเข้าไปเปิดแก้ไฟล์รองบรรทัดที่ 770 เปลี่ยนเลข 10 เป็น 25 ได้เลย

## 4. พารามิเตอร์สำคัญของระบบขับเคลื่อน (Drive & Navigation Parameters)
เป็นพารามิเตอร์ควบคุมพฤติกรรมการเคลื่อนที่และทิศทางของหุ่นยนต์ตัดหญ้าซึ่งตกหล่นไปในคู่มือฉบับเก่า:

* **`BNO_OFFSET_DEG` (ค่าชดเชยทิศเหนือของเข็มทิศ IMU) [จูนในไฟล์รอง `teleop_stm.py` เท่านั้น]:**
  * *คำอธิบาย:* ใช้ระบุองศาชดเชยเข็มทิศชี้ทิศเหนือของบอร์ด IMU BNO055 เพื่อปรับระดับทิศหน้าของหุ่นยนต์ตัดหญ้าในโปรแกรมให้หมุนหันตรงตามทิศเหนือจริงในสนาม GPS
  * *ค่าเริ่มต้น:* `-190.0` (ที่บรรทัด 20 ของ `teleop_stm.py`)
  * *การจูน:* เป็นตัวแปร Global Constant ในภาษา Python **ต้องเปิดเข้าไปแก้ไขบรรทัดที่ 20 ในไฟล์รอง `teleop_stm.py` โดยตรง**

## 6. การจูนพารามิเตอร์ระบบกล้อง AI (AI Vision Parameter Tuning Guide)
การตรวจจับคน สิ่งมีชีวิต และขอบเขตความลึกเพื่อสั่งหยุดหุ่นยนต์ ใช้ YOLOv11 และ Depth Map ของกล้อง RealSense D435i ประมวลผลในโหนด [**vision_node.py**](file:///home/nhio/ros2_ws/src/robot_bridge/robot_bridge/vision_node.py) โดยสามารถจูนค่าเพื่อปรับประสิทธิภาพการตรวจจับและความปลอดภัยได้ดังนี้:

### 1) พารามิเตอร์หลักทางระบบ (ROS2 Launch Parameters)
พารามิเตอร์กลุ่มนี้มีประกาศใช้งานอยู่ทั้งในไฟล์หลัก Launch และโค้ดแกนหลัก Python:
* **จุดที่ต้องแก้ไข:** **[จูนผ่านไฟล์ Launch ได้เลย]** ให้ทำการจูนที่ไฟล์เปิดระบบหลัก [**hardware_bringup.launch.py**](file:///home/nhio/ros2_ws/src/robot_bridge/launch/hardware_bringup.launch.py) (บรรทัด 18-22) เพียงจุดเดียวเท่านั้น ระบบจะดึงค่านี้ไปเขียนทับตัวแปรในโค้ด Python ให้โดยอัตโนมัติ
* **`model_path` (ค่าเริ่มต้นในโค้ด: `'yolo11n.pt'`):**
  * *คำอธิบาย:* ชื่อไฟล์น้ำหนักโมเดล YOLO ที่ใช้สแกนหาวัตถุ
  * *การจูน:* สามารถเปลี่ยนไปใช้โมเดลที่แม่นยำขึ้น เช่น `'yolo11s.pt'` หรือไฟล์โมเดลที่ผ่านการเทรนเองได้
* **`stop_distance` (ค่าเริ่มต้นในโค้ด: `4.0` เมตร):**
  * *คำอธิบาย:* ระยะปลอดภัยของ AI ตรวจจับคน/หมา/แมว หากพบใกล้กว่าค่านี้จะสั่งหยุดเบรกฉุกเฉิน
  * *การจูน:* ลดค่าลงได้หากต้องการให้หุ่นยนต์วิ่งเข้าเบียดขวางเป้าหมายได้ชิดขึ้น (เช่น `2.0` - `3.0` เมตร)
* **`fence_distance` (ค่าเริ่มต้นในโค้ด: `1.5` เมตร):**
  * *คำอธิบาย:* ระยะตรวจจับวัตถุแปลกปลอมแบบรวมก้อนความลึก (Depth Fence) ซึ่งจะจับวัตถุทุกชนิดที่นูนขึ้นมาจากพื้นดิน
  * *การจูน:* ปรับตามสภาพพื้นที่เพื่อป้องกันล้อหน้าเบียดชนขอบปูน กระถาง หรือดินสไลด์

### 2) พารามิเตอร์ควบคุมอัลกอริทึมในโค้ด (Internal Algorithm Tuning)
สำหรับพารามิเตอร์เหล่านี้ไม่มีการเปิดระบบพารามิเตอร์ลิงก์ไว้กับ Launch File:
* **จุดที่ต้องแก้ไข:** **[จูนในไฟล์รอง (Python) เท่านั้น]** ผู้พัฒนาต้องเปิดไฟล์โค้ดหลัก [**vision_node.py**](file:///home/nhio/ros2_ws/src/robot_bridge/robot_bridge/vision_node.py) เพื่อเข้าไปแก้ไขตัวเลขบรรทัดโค้ดโดยตรง:
* **ค่าความมั่นใจในการตรวจจับของ YOLO (`conf=0.4`):**
  * *คำอธิบาย:* เกณฑ์คะแนนขั้นต่ำที่ยอมรับว่าวัตถุที่เห็นคือเป้าหมายคน/สัตว์จริง
  * *การจูน:* เพิ่มขึ้นเป็น `0.5` หรือ `0.6` หากต้องการแก้ปัญหา AI ตรวจจับใบไม้หรือเสาเป็นคน (ลดปัญหาเบรกฉุกเฉินหลอก)
* **ขนาดภาพอินพุตกล้องของโมเดล (`imgsz=320`):**
  * *คำอธิบาย:* ขนาดพิกเซลที่บีบอัดก่อนส่งป้อนให้ YOLO AI สแกน
  * *การจูน:* ตัวเลข `320` ถูกตั้งไว้เพื่อให้รันเร็วจานลื่นบน Raspberry Pi 5 หากนำไปรันบนคอมพิวเตอร์การ์ดจอสูง สามารถปรับเป็น `640` เพื่อให้ตรวจจับเป้าหมายขนาดจิ๋วที่ระยะไกลได้แม่นยำขึ้นมาก
* **จำนวนเฟรมยืนยันวัตถุ AI (`CONFIRM_FRAMES_AI = 5`):**
  * *คำอธิบาย:* จำนวนเฟรมภาพติดต่อกันที่ต้องเจอ คน/หมา/แมว ก่อนสั่งส่งเบรกจริง (5 เฟรม = ~0.33 วินาที ที่ความเร็ว 15 FPS)
  * *การจูน:* ทำหน้าที่กรองแสงสะท้อนหรือภาพกะพริบ ปรับลดลงเหลือ `2` - `3` หากต้องการเบรกเร็วดั่งใจคิด หรือเพิ่มเป็น `8` - `10` เพื่อความนุ่มนวลในการเคลื่อนที่
* **จำนวนเฟรมยืนยันกำแพงความลึก (`CONFIRM_FRAMES_FENCE = 5`):**
  * *คำอธิบาย:* จำนวนเฟรมภาพติดต่อกันที่สัญญาณ Depth รุกล้ำขอบเขตเข้ามา
  * *การจูน:* ช่วยกรองเม็ดฝุ่นหรือแสงสะท้อนแดดจัดบนหญ้าเปียกไม่ให้ขวางรถตัดหญ้า
* **เกณฑ์ความกว้างจำนวนพิกเซลสั่งชนกล้อง (`len(close_points) > 1500`):**
  * *คำอธิบาย:* จำนวนจุดพิกเซลความลึกที่พบใกล้เกินระยะ `fence_distance` ภายในกรอบ ROI สแกน
  * *การจูน:* เพิ่มขึ้นเป็น `2500` - `3000` เพื่อข้ามการเบรกฉุกเฉินเมื่อเจอต้นหญ้าเดี่ยวๆ ชูช่อสูง หรือลดลงเหลือ `800` เพื่อความละเอียดปลอดภัยสูงสุด
* **ระยะหน่วงหยุดของเบรกขั้นต่ำ (`MIN_STOP_DURATION = 0.5` วินาที):**
  * *คำอธิบาย:* เวลาดึงเบรกค้างขั้นต่ำหลังความปลอดภัยถูกสั่งให้หยุด (ระบบป้องกันเบรกสั่นกระตุก)
  * *การจูน:* เพิ่มเป็น `1.0` วินาทีเพื่อปกป้องชุดมอเตอร์เกียร์ล้อจากความเสียหายของการเบรกถี่
* **รายการพืชที่กดยกเว้นการชน (Plant Bypass List) [บรรทัด 133-134]:**
  * *คำอธิบาย:* ระบบระบุให้คลาส `'potted plant'` (ไม้กระถาง/พืช) ถือเป็นข้อยกเว้น โดยระบบจะหักลบพื้นที่ Bounding Box ของต้นไม้ออกจากการคำนวณ Depth Fence ป้องกันไม่ให้รถตัดหญ้าเบรกหัวทิ่มเมื่อวิ่งเฉียดใบไม้
  * *การจูน:* สามารถเพิ่มคลาสพืชในโมเดลคัสตอมอื่นๆ หรือคลาสธรรมชาติเข้าไปในโค้ดเพิ่มเพื่ออำนวยความสะดวกในการวิ่งผ่านธรรมชาติได้

### 3) ระบบกำแพงความลึกหน้ารถตรง (Depth Safety Fence - ROI & Pixel Tuning)
แนวกั้นความปลอดภัยกล้องความลึก **[Depth Safety Fence]** ในไฟล์ [**vision_node.py**](file:///home/nhio/ros2_ws/src/robot_bridge/robot_bridge/vision_node.py) ทำงานโดยกวาดดูแนวความลึกหน้ารถตรงและสั่งหยุดทันทีหากมีวัตถุนูนใดๆ โผล่เข้ามาขวางทาง (เช่น หิน ก้อนดิน ถังน้ำ โครงเหล็ก) โดยระบบความปลอดภัยตัวนี้ใช้พิกัดกล้องควบคุมพื้นที่สแกนดังนี้:

| vision_node.py : |
| :--- |
| `roi_x1, roi_x2 = int(w * 0.0), int(w * 1.0)  # แนวนอน (ซ้ายสุดไปขวาสุด)`<br>`roi_y1, roi_y2 = int(h * 0.2), int(h * 0.65)  # แนวตั้ง (บนลงล่าง)` |

* **จุดที่ต้องแก้ไข:** **[จูนในไฟล์รอง (vision_node.py) เท่านั้น]** การทำงานของ Depth Fence จะจำกัดกรอบพื้นที่ในหน้าจอภาพเพื่อกรองสิ่งแวดล้อมดังนี้:
  * **ขอบบน (`roi_y1 = int(h * 0.2)`):** บีบขอบบนลงมา 20% เพื่อตัดท้องฟ้า ทิวทัศน์ หรือต้นไม้ที่อยู่สูงพ้นตัวหุ่นยนต์ออกไป จะได้ไม่จับวัตถุด้านบนมาเบรก
  * **ขอบล่าง (`roi_y2 = int(h * 0.65)`):** **(จุดวิเคราะห์สำคัญ)** บีบขอบล่างขึ้นไปสูงกว่าขอบจริง 35% เพื่อหลีกเลี่ยงไม่ให้กล้องเผลอก้มไปมองเห็น **"ฝากระโปรงรถตัวเอง หรือพื้นดินใต้ล้อตรงหน้าตรง"** ซึ่งหากไม่บีบขอบนี้ขึ้น รถตัดหญ้าจะอ่านเจอตัวรถตัวเองและสั่งล็อกเบรกถาวรทันที
  * **การจูนหน้างาน:**
    * หากรถเบรกหลอกเพราะสแกนเจอขอบฝาครอบบอดี้ของตัวเอง ➡️ ให้ลดค่า `0.65` เหลือ **`0.60`** หรือ **`0.58`** เพื่อยกกรอบล่างพ้นตัวรถขึ้นไป
    * หากรถวิ่งชนกระถางต้นไม้หรือขอบปูนกั้นหญ้าที่สูงระดับต่ำเตี้ยดิน ➡️ ให้เพิ่มค่า `0.65` เป็น **`0.70`** เพื่อเปิดตามองมุมก้มกวาดพื้นให้ต่ำลงกว่าเดิม

## 7. เครื่องมือคำสั่งตรวจสอบข้อมูลและสืบค้น (CLI Sniffing & Diagnosis)
ในการทดสอบระบบและเฝ้าระวังสัญญาณจริงบนตัวรถตัดหญ้า คุณสามารถเปิดใช้งานคำสั่งสากลของ ROS2 เพื่อ "ดักฟัง" หรือ "สแกนหาข้อบกพร่อง" ได้ทันทีผ่าน Terminal:

### 1. การสืบหาและรายการหัวข้อส่งสัญญาณทั้งหมด (Topic Hunting)
เพื่อสแกนหาว่า ณ ปัจจุบัน มีช่องสัญญาณ Topic ตัวไหนกำลังพูดคุยกันอยู่ในหุ่นยนต์บ้าง:
```bash
ros2 topic list
```

### 2. การดักฟังดูข้อมูลพารามิเตอร์แบบ Real-time (Sniffing Data)
หากผู้พัฒนาต้องการจับเข่าส่องตัวเลขข้อมูลสดๆ ที่ส่งออกมารอบต่อวินาที:
* **ดักฟังความเร็วที่ถูกสั่งให้หุ่นตัดหญ้าวิ่งอัตโนมัติ:**
  ```bash
  ros2 topic echo /cmd_vel_filtered
  ```
* **ดักฟังสถานะความปลอดภัยของแนวรั้ว Geofence (0 = ปลอดภัย, 1 = ทะลุรั้วให้สั่งเบรก):**
  ```bash
  ros2 topic echo /emergency_stop
  ```
* **ดักฟังข้อความสั่งหยุดรถฉุกเฉินจากกล้อง AI Vision / เลเซอร์ LiDAR:**
  ```bash
  ros2 topic echo /cmd_emergency
  ```
* **ดักฟังพิกัดดาวเทียมดิบ ละติจูด ลองจิจูด และชนิดสัญญาณดาวเทียม:**
  ```bash
  ros2 topic echo /fix
  ```

### 3. ตรวจสอบความถี่ความแรงของสัญญาณ (Frequency & Rate Check)
ใช้เพื่อเช็คว่าโหนดปลายทางเกิดอาการค้างหรือส่งข้อมูลขาดช่วงช้าเกินไปหรือไม่:
* **เช็คความถี่การทำงานของเลเซอร์สแกน LiDAR (ปกติควรส่งที่ความถี่ 7-10 Hz):**
  ```bash
  ros2 topic hz /scan
  ```
* **เช็คความเร็วการตอบกลับสัญญาณของ EKF Sensor Fusion (ปกติควรส่งที่ความถี่ 10-20 Hz):**
  ```bash
  ros2 topic hz /odometry/global
  ```

### 4. ตรวจสอบเส้นทางสอดรับภายในโหนดปลายทาง (Node Inspection)
ต้องการส่องข้อมูลเชิงลึกของโหนดควบคุมหลัก `teleop_stm` ว่ามันรับข้อมูลจากโหนดไหนบ้างและส่งให้โหนดไหนอยู่:
```bash
ros2 node info /teleop_stm
```

### 5. การเรียกใช้และดึงค่า Parameter นอกสถานที่
ต้องการสอบถามสมองหุ่นยนต์ว่า ขณะนี้เราเปิดใช้งานระบบข้ามความปลอดภัย (Bypass Mode) อยู่หรือไม่:
```bash
ros2 param get /teleop_stm bypass_safety
```
